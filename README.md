# Анализ музыкальных треков Spotify

Исследование ~114 000 треков Spotify: какие аудио-характеристики связаны
с популярностью трека и чем различаются музыкальные жанры.

Итоговый проект по дисциплине «Анализ данных на Python».

## Данные

**Источник:** [Spotify Tracks Dataset](https://www.kaggle.com/datasets/maharshipandya/spotify-tracks-dataset)
(Kaggle) — 114 000 треков, 114 жанров, 20 признаков на трек.

Каждый трек описан аудио-характеристиками, которые Spotify рассчитывает
алгоритмически:

| Признак | Описание |
|---|---|
| `danceability` | танцевальность, 0–1 |
| `energy` | энергичность, 0–1 |
| `valence` | позитивность настроения, 0–1 |
| `acousticness` | акустичность, 0–1 |
| `instrumentalness` | отсутствие вокала, 0–1 |
| `speechiness` | доля речи, 0–1 |
| `liveness` | «живое» исполнение, 0–1 |
| `loudness` | громкость, dB |
| `tempo` | темп, BPM |
| `popularity` | популярность, 0–100 |

Исходный датасет содержит аудио-фичи, но не метаданные артиста, поэтому
он **обогащается через внешний API** (жанры, страна и теги артиста).

## Что сделано

- **Сбор и обогащение.** Готовый датасет дополнен данными об артистах
  через музыкальный API. Запросы выполняются асинхронно (`aiohttp`),
  с ограничением параллелизма и кэшированием результата.
- **Очистка.** Обработка пропусков (медиана / мода), удаление дубликатов,
  исправление аномалий (нулевой темп, невалидный музыкальный размер),
  обработка выбросов методом IQR, создание новых признаков.
- **Анализ.** Разведочный анализ и проверка четырёх гипотез
  статистическими тестами (корреляции Спирмена и Пирсона,
  критерии Манна–Уитни и Краскела–Уоллиса).
- **Визуализация.** Гистограмма, boxplot, тепловая карта корреляций,
  диаграмма рассеяния с линией тренда, столбчатая диаграмма.

## Основные результаты

- Танцевальность почти **не** определяет популярность трека
  (ρ ≈ 0.03): популярность зависит от внешних факторов, а не звучания.
- Жанры сильно различаются по энергичности
  (metal ≈ 0.84 против classical ≈ 0.20).
- Треки с пометкой *explicit* в среднем популярнее (медиана 38 против 34).
- Позитивность и танцевальность связаны положительно (r ≈ 0.48).

## Структура

```
spotify_analysis/
├── notebook.ipynb     # основной ноутбук с анализом
├── requirements.txt   # зависимости
├── .env.example       # шаблон ключей Spotify API
├── src/
│   └── enrich.py      # асинхронное обогащение данных через API
└── data/
    ├── raw/           # исходный датасет
    └── processed/     # обогащённые данные
```

## Запуск

```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
jupyter lab notebook.ipynb
```

Обогащение через Spotify API требует ключей (`.env` по образцу
`.env.example`). Без ключей оно автоматически использует MusicBrainz —
открытую музыкальную базу, не требующую авторизации.

## Стек

Python, pandas, NumPy, matplotlib, seaborn, scikit-learn,
SciPy, statsmodels, aiohttp.
